# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목

"짐봐줘 - 여행자 짐 보관 매칭 시스템"

## 2. 시나리오 (5~10줄)

프로그램은 여행자가 공항/여행지 근처의 일반 가정집에 짐을 맡길 수 있도록 연결해주는 콘솔 기반 매칭 시스템입니다. 사용자는 보관자(집 주인) 또는 여행자로 등록할 수 있습니다. 여행자는 짐의 종류(캐리어, 강아지, 캐리어+강아지 등)와 보관 희망 시간대를 입력하면, 시스템이 해당 조건에 맞는 보관 가능 집을 안내합니다. 보관 시간에 따라 요금이 자동 계산되며, 마트 짐·반려동물·캐리어 등 짐 종류별로 추가 요금이 산정됩니다. 최종적으로 예약 정보와 예상 결제 금액을 출력합니다.

## 3. 예상 기능 및 메뉴 (최소 5개)

```
============================
   🧳 짐봐줘 매칭 시스템
============================
1 --- 여행자 정보 입력
2 --- 짐 종류 및 보관 시간 입력
3 --- 예상 요금 계산
4 --- 보관 가능 집 안내
5 --- 예약 확인서 출력
0 --- 종료
============================
```

## 🟨 [3차 과제: V3.0] 무한 루프와 메뉴 시스템 (반복문) - 향후 작성 예정
### **✨3차 과제 업데이트 내용:**
  * 내용.3차 과제는 2차에서 만든 짐봐줘에 반복문과 함수를 추가해서 프로그램이 계속 실행되고 기능별로 모듈화되도록 구현했습니다.

while True 무한 루프로 메인 메뉴를 출력하고, 사용자가 종료를 선택할 때만 break로 프로그램이 끝나도록 구성

2차에서 순서대로 실행되던 코드를 get_traveler_info(), select_luggage(), calculate_fee(), match_house(), print_receipt(), add_reservation(), show_reservations(), show_analysis() 총 8개의 함수로 분리해서 각 기능이 독립적으로 동작하도록 구현

calculate_fee(hours, extra_fee), match_house(needs_pet) 등 매개변수를 받아 처리하는 함수 구현

get_traveler_info()는 리스트, calculate_fee()는 딕셔너리, match_house()는 보관자 정보를 return으로 반환하도록 구현

reservations, available_houses 등 프로그램 전체에서 공유하는 데이터는 전역 변수로 선언하고, BASE_RATE 같은 계산 중간값은 함수 내부 지역 변수로 처리, add_reservation() 안에서 global 키워드 사용해서 전역 리스트에 예약 데이터 추가

예약 입력 외에 전체 예약 목록 조회와 평균/최고/최저 요금 분석 기능도 추가
    
### **🤖 AI 파트너십 과정**
 1.while True 메뉴 구조 설계
프롬프트 요약: 2차 코드를 while True 루프 기반 메뉴로 바꾸고 싶은데 어떻게 구성하면 좋을지 물어봄
적용 내용: 1.예약입력 2.예약조회 3.분석 4.종료 메뉴 구성, 종료 선택 시에만 break로 탈출하는 구조로 적용
함수 분리 기준 잡기
프롬프트 요약: 기존 코드를 함수로 나누려는데 어디까지 쪼개야 하는지 기준을 모르겠다고 질문
적용 내용: "한 함수가 하나의 역할만 한다"는 원칙으로 입력/계산/매칭/출력/저장을 각각 별도 함수로 분리
global 키워드 사용법
프롬프트 요약: 함수 안에서 reservations 리스트에 데이터를 추가하려는데 오류가 난다고 질문
적용 내용: 함수 내부에서 전역 변수를 수정할 때 global 선언이 필요하다는 걸 이해하고 add_reservation() 안에 global reservations 추가
    
### **🛠️ Troubleshooting & 기술 회고:**
  문제: 함수 안에서 reservations.append() 실행 시 UnboundLocalError 발생
원인: 파이썬은 함수 내부에서 전역 변수를 수정하려 하면 지역 변수로 인식해서 오류 발생
해결: add_reservation() 함수 맨 위에 global reservations 선언 추가
문제: print_receipt() 호출 시 인자 순서를 잘못 넣어서 출력이 뒤섞임
원인: 매개변수가 많다 보니 호출할 때 순서를 헷갈려서 잘못 전달
해결: 함수 정의부와 호출부를 나란히 놓고 순서 하나씩 맞춰가며 수정
문제: 메뉴에서 잘못된 번호 입력 시 프로그램이 그냥 넘어가버림
원인: else 처리만 하고 다시 입력받는 루프를 안 넣음
해결: while True + break 구조로 올바른 입력이 들어올 때만 다음으로 넘어가도록 수정
