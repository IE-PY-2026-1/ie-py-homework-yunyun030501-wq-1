# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목

"짐봐줘 - 여행자 짐 보관 매칭 시스템"

## 2. 시나리오 (5~10줄)

프로그램은 여행자가 공항/여행지 근처의 일반 가정집에 짐을 맡길 수 있도록 연결해주는 콘솔 기반 매칭 시스템입니다. 사용자는 보관자(집 주인) 또는 여행자로 등록할 수 있습니다. 여행자는 짐의 종류(캐리어, 강아지, 캐리어+강아지 등)와 보관 희망 시간대를 입력하면, 시스템이 해당 조건에 맞는 보관 가능 집을 안내합니다. 보관 시간에 따라 요금이 자동 계산되며, 마트 짐·반려동물·캐리어 등 짐 종류별로 추가 요금이 산정됩니다. 최종적으로 예약 정보와 예상 결제 금액을 출력합니다.

## 3. 예상 기능 및 메뉴 (최소 5개)

```
============================
   🧳 짐봐줘 매칭 시스템
============================
1 --- 여행자 정보 입력
2 --- 짐 종류 및 보관 시간 입력
3 --- 예상 요금 계산
4 --- 보관 가능 집 안내
5 --- 예약 확인서 출력
0 --- 종료
============================
```

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
이번 과제는 1차에서 만든 짐봐줘에 리스트와 조건문을 추가해서 데이터를 저장하고 상황에 따라 다르게 출력되도록 구현했습니다.
input()으로 여행자 이름, 여행지, 연락처, 짐 종류, 보관 시간을 입력받아 traveler_info 리스트에 저장
luggage_menu, available_houses, extra_fees를 리스트로 만들어서 인덱스로 접근하도록 구성
if-elif-else로 보관 시간에 따라 할인율을 3단계로 나눠서 자동 계산 (8시간 이상 20%, 4시간 이상 10%, 그 외 0%)
짐 종류 번호(1~4)에 따라 추가 요금이랑 안내 문구가 다르게 출력되게 조건 분기 처리
and/or 연산자 써서 반려동물 포함 여부 판단하고, 프리미엄 케어 조건도 만들었음
for문이랑 조건문 같이 써서 available_houses에서 조건 맞는 보관자 자동으로 매칭

    
### **🤖 AI 파트너십 과정**
내용 1: 리스트 구조 어떻게 짤지 물어봤음 
프롬프트 요약: "우리 전에 했던 짐봐줘에서 목록들을 리스트로 관리하려고 해. 1학년 수준에서 이해하기 쉬운 방식으로 어떻게 구조하면 좋을까?"
적용 내용:

여행자 정보를 변수 여러 개로 따로 관리하던 걸 traveler_info 리스트 하나에 .append()로 순서대로 넣는 방식으로 바꿈
보관자 정보는 [이름, 지역, 펫가능여부, 평점] 이렇게 묶어서 리스트 안에 리스트로 저장하는 구조 사용
짐 종류 요금을 extra_fees 리스트에 인덱스 순서 맞춰 넣어서, 선택 번호만 알면 요금이랑 안내문구 바로 꺼낼 수 있게 함

내용 2: 조건문이랑 논리 연산자 어떻게 쓸지 물어봄
프롬프트 요약: "보관 시간에 따라 요금을 다르게 하고 싶고, 반려동물 있을 때만 특정 조건 주고 싶어. if-elif-else랑 and/or 둘 다 쓰는 상황을 자연스럽게 만들어줘."
적용 내용:

보관 시간 구간을 8시간 이상 / 4시간 이상 / 그 미만으로 나눠서 if-elif-else로 할인율 다르게 적용
needs_pet 변수를 or 연산자로 만들어서 2번이나 4번 선택하면 반려동물 포함으로 처리되게 함
is_premium은 needs_pet and (hours >= 4) 조건으로, 두 조건 다 맞을 때만 프리미엄 배정되게 구현
보관자 매칭할 때도 if needs_pet and house[2] 조건 써서 펫 가능한 집 먼저 매칭되도록 함
